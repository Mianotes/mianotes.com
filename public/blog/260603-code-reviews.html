Research workflow

Share AI skills, prompts, and research findings with Mianotes

A practical workflow for using one shared computer as the place where a team stores source material, reusable prompts, agent handoffs, and useful LLM output.

Mianotes
5 Jun 2026

Imagine you are the software architect on the Argentinc team. A client asks a broad question: how can AI improve sales on our ecommerce website?

You need to look at customer journeys, merchandising, search, recommendations, support conversations, email campaigns, product data, checkout friction, analytics, and competitor examples. Your developers, designers, and product managers all need to see the same research. Codex or Claude can help analyse the material, but only if the work does not disappear into temporary chats.

The goal is to share more than files. You want the team's AI skills, useful prompts, research findings, source links, and decisions to live somewhere everyone can browse and future agents can read before they start working.

For this example, we will use the most popular AI desktop computer right now: a Mac mini named research.local. The same pattern works with any server on your local network. Install Mianotes once, point everyone at the same browser address, connect your AI agent through MCP, and start saving the useful parts of the work as Markdown notes.

Team memory

Install Mianotes once, then give developers, designers, and product managers one shared place to collect skills, prompts, findings, and decisions.

Project workspace

In Mianotes, a workspace is the top-level place where a project's notes live. This one is called AI Agents.

1. Download the macOS package

On the Mac mini, download the latest macOS package from GitHub.

Open mianotes.pkg, follow the installer, then open the dashboard at http://research.local:8201. If your network does not resolve research.local, use the Mac mini's LAN address instead, for example http://192.168.x.x:8201.

The package installs Mianotes as two launchd services:

  • The API runs on port 8200.
  • The dashboard runs on port 8201.
  • Runtime tools such as ffmpeg, ffprobe, and ripgrep are bundled; Apple Silicon builds also include Tesseract OCR.
  • The packaged MCP command is available as mianotes-mcp.

If something looks wrong, run:

Terminal 
mianotes doctor
mianotes status
mianotes logs

2. Create the research workspace

Open Mianotes at http://research.local:8201, create the first admin account, then add a workspace for the project. Think of a workspace as the project area that holds the notes, folders, sources, and decisions for one piece of work.

1. Choose a workspace folder

Choose a folder on the Mac mini that the user running the Mianotes service can read and write, for example /Users/mianotes/Desktop/research. People using Mianotes from other machines do not need direct filesystem access to this folder; they use the dashboard in the browser.

2. Add folders inside Mianotes

Create folders named Sources, Findings, Opportunities, Decisions, and Handoffs.

3. Keep one workspace per project

For this article, the workspace is AI Agents. The team can search it by name from Codex, Claude, or the web app.

3. Connect agents to Mia

This is one setup with two small actions: create the API key in Mianotes, then add the packaged MCP command to Codex or Claude Code.

Create the API key in Mianotes

In Mianotes, go to Settings and create an API key. Mianotes writes the API key to its service environment file, so mianotes-mcp can load it without you pasting secrets into prompts.

Do not paste the API key into Codex or Claude.

The packaged mianotes-mcp command reads the key from Mianotes' environment file and exchanges it for a short-lived agent session.

Add Mianotes to Codex or Claude Code

The dashboard is available to everyone on the local network. MCP is different: Codex or Claude Code launches mianotes-mcp on the same machine where the agent runs. For this setup, run agents on the Mac mini, or install the MCP command on each developer machine and point it at the Mac mini API.

Codex

Terminal 
command -v mianotes-mcp
codex mcp add mianotes --env MIANOTES_CLIENT_NAME=Codex -- mianotes-mcp
codex mcp list

Claude Code

Terminal 
claude mcp add-json mianotes '{"type":"stdio","command":"mianotes-mcp","env":{"MIANOTES_CLIENT_NAME":"Claude"}}'
claude mcp get mianotes

Inside Claude Code, run /mcp. Mianotes should appear as a connected server.

Start a new Codex or Claude Code session after adding the MCP server. Then test it with:

LLM
Mia, show me the folders in the AI Agents workspace.
Worked for 13s

Mia didn't find a workspace named AI Agents.

4. Start sending research to Mia

For links, Codex and Claude can use MCP. For local file bytes, ask the agent to use the Mianotes REST file upload endpoint. The MCP server intentionally stays thin and does not read arbitrary files directly.

Here are the prompts I would give the agent as the Argentinc architect leading the ecommerce AI research.

Import public source material

For example, collect a personalisation report, an AI product discovery article, and a shopping assistant case study, then ask the agent to save them in Mia.

LLM
Import the three source links listed above into Mia(workspace: AI Agents, folder: Sources). Create useful titles and tag them with "ai", "sales", and "ecommerce".
Worked for 24s

Done. I added three source notes to AI Agents in Sources, with titles, tags, and source links preserved.

Upload local documents

When the source is a local PDF, deck, spreadsheet, screenshot, or audio file on the Mac mini, be explicit that the agent should use the REST upload endpoint and keep secrets out of the output.

LLM
Upload /Users/mianotes/Desktop/research/customer-support-themes.pdf to Mia(workspace: AI Agents, folder: Sources). Use the REST file upload endpoint and do not print secrets.
Worked for 41s

Uploaded. Mia created a source note in Sources and queued the parsing job. The note will update when the PDF text has been extracted.

Save useful LLM output

When Codex or Claude produces something useful, save it immediately. This is the part that turns a chat into project memory.

LLM
Save the market map you just created in Mia(workspace: AI Agents, folder: Findings). Use the title "Market map: AI features that can improve ecommerce sales".
Worked for 18s

Added to Mianotes with the title Market map: AI features that can improve ecommerce sales.

5. A realistic architect workflow

Here is the full pattern I would use for the ecommerce AI project.

1. Collect evidence

Upload analytics exports, support themes, product catalog notes, competitor screenshots, PDFs, and research links into Sources.

2. Ask the agent to compare patterns

Ask Codex or Claude to read the workspace and identify repeated opportunities: better search, guided selling, recommendations, abandoned cart recovery, customer support deflection, and product copy generation.

3. Save decisions

When the team agrees on priorities, save the final decision notes into Decisions so the next agent starts from the current truth.

A useful prompt for the analysis step looks like this:

LLM
Before answering, get context from Mia(workspace: AI Agents, folder: Sources). I am preparing an architecture proposal for AI features that could improve sales on our ecommerce website. Identify the top five opportunities, explain ownership, list the data needed, call out implementation risk, and save your answer in Mia(workspace: AI Agents, folder: Opportunities).
Worked for 1m 12s

I found five repeated opportunities across the sources: product discovery, guided selling, search relevance, abandoned cart recovery, and support deflection.

I saved the recommendation in AI Agents under Opportunities.

6. What the team gets back

After a few sessions, the workspace becomes more useful than the chat history. Developers can search for implementation constraints. Designers can read customer pain points. Product managers can review opportunities and decisions. Future agents can read the same notes before changing anything.

  • Source documents are converted into Markdown notes.
  • Agent findings are saved with titles, folders, and tags.
  • Everyone can browse the research from research.local.
  • Codex and Claude can retrieve the same project memory through MCP.

The habit is simple: put evidence into Mia, ask the agent to reason over it, then save the useful output back into Mia.

For Argentinc's ecommerce AI project, that means the research does not live in one architect's notebook or one LLM thread. It becomes a shared working memory for the product team, design team, engineering team, and the next agent that joins the work.
